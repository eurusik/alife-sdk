# Core Reference

Use this track when you are integrating the runtime shell behind `@alife-sdk/core`.

These pages are the contracts you usually need once you stop prototyping and start wiring real runtime behavior.

## Browse by task

- "I need to observe runtime changes through events" -> [Core Events Reference](/docs/reference/core/events)
- "I need to decide what belongs in the kernel" -> [Core Plugins Reference](/docs/reference/core/plugins)

## What this track covers

- how events are queued and flushed
- what plugins actually own and when their hooks fire
- what the SDK expects from an entity shape
