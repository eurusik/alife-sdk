# AI Reference

Use this track when you are integrating observed, online NPC behavior through `@alife-sdk/ai`.

These pages explain how the online driver, host context, and shared AI subsystems fit together once an NPC is no longer only an offline simulation record.

## What this track covers

- one driver per online NPC
- `INPCContext` as the bridge into your engine
- perception as the live information layer
- cover as the shared tactical positioning layer
